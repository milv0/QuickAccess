# Architecture Documentation Rule

## 트리거

다음 변경이 발생하면 `ARCHITECTURE.txt`를 반드시 업데이트한다:

- 새 파일/폴더 생성 또는 삭제
- 파일/폴더 이름 변경 또는 이동
- 새로운 기능 추가 (PR 단위)
- 런처 타입 추가/변경
- 단축키 추가/변경
- 설정 파일 경로/구조 변경

## 업데이트 범위

변경된 내용에 해당하는 섹션만 수정:

| 변경 종류 | 업데이트할 섹션 |
|-----------|----------------|
| 파일/폴더 구조 | "프로젝트 구조" |
| 기능 추가/제거 | "핵심 기능" |
| 단축키 변경 | "단축키" |
| 런처 변경 | "런처 타입별 동작" |
| 권한 변경 | "권한 요구사항" |
| 빌드/배포 변경 | "개발 명령어" 또는 "배포 현황" |

## 형식

- 기존 ARCHITECTURE.txt의 스타일을 유지
- 간결하게 한 줄 설명
- 파일 트리는 실제 구조와 일치시킬 것
- 불필요한 설명 추가하지 말 것

## 예시

파일 `Sources/Chap/Launchers/SafariLauncher.swift` 추가 시:
→ "프로젝트 구조"에 `│   │   ├── SafariLauncher.swift` 추가
→ "핵심 기능"에 `Safari → ...` 추가
→ "런처 타입별 동작"에 Safari 설명 추가
