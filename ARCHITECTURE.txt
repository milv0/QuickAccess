Chap Project Structure
======================

## 앱 소개

Chap — 키보드 단축키 한 번으로 웹사이트·앱·폴더를 원하는 화면, 원하는 크기로 즉시 여는 macOS 메뉴바 런처.


## 프로젝트 구조

Sources/

│   ├── AppDelegate.swift          ← 라이프사이클, 메뉴, 핫키 (CGEvent tap)

│   │   ├── FinderLauncher.swift   ← Finder (단일 AppleScript)

│   │   ├── LauncherUtils.swift    ← 공용 AX API, retryResize, showAlert
│   │   ├── ResizeLogger.swift     ← CSV 성능 로그
│   │   └── GhostWindow.swift      ← 리사이즈 미리보기 윈도우
│   └── Views/                     ← UI
│       ├── DesignSystem.swift     ← 색상, 폰트, 컴포넌트 스타일

│       └── WelcomeView.swift      ← 첫 실행 가이드

│   ├── Models.swift               ← Site, Config, LaunchType, Defaults

## 핵심 기능

- 글로벌 단축키 (⌥Q, ⌥커스텀키, ⌥,) — CGEvent tap
- URL → Chrome --app 모드 + AX API 리사이즈
- App → 실행 + AX API 리사이즈
- Finder → 폴더 열기 + 리사이즈 (단일 AppleScript, 딜레이 없음)
- Shell → 스크립트 실행 (리사이즈 없음)
- 멀티 모니터 지원 + 미니맵
- 클릭 편집 + 자동 저장 (Enter/포커스 이동 시)
- 사이트별 커스텀 단축키 (optional, 중복 체크)
- Import (파일/Paste JSON/드래그앤드랍) + Export → ~/Downloads
- Config 자동 백업 (.bak)
- 순서 변경/삭제 시 자동 저장
- Restart / Uninstall (메뉴바/설정 File 메뉴)
- 리사이즈 성능 로그 (CSV)
- 접근성 권한 자동 재시도 + 권한 제거 감지
- User Guide (⌘?)


## 단축키

글로벌 (앱 외부, ⌥ Option):
  ⌥Q         메뉴 열기
  ⌥(커스텀키)  사이트 실행 (사이트별 설정, optional)
  ⌥,         설정 열기

설정 창 내 (⌘ Command):
  ⌘1~9   사이트 선택
  ⌘S     저장
  ⌘?     가이드
  Enter  저장 + 편집 모드 해제


## 런처 타입별 동작

  URL:     Chrome --app 모드 실행 → AX API로 새 윈도우 감지 → position/size 설정
  App:     NSWorkspace.openApplication → AX API로 포커스 윈도우 → position/size 설정
  Finder:  단일 AppleScript로 열기+리사이즈 동시 (딜레이 없음)
  Shell:   Process로 스크립트 실행 (리사이즈 없음)

리사이즈 전략:
  URL (Chrome 실행 중):  AX API 폴링 50ms 간격 (윈도우 수 증가 감지, 최대 2초)
  URL (콜드 스타트):     AX API 폴링 100ms 간격 (최대 6초)
  App (실행 중):         AX API 폴링 50ms 간격 (포커스 윈도우 감지, 최대 1.5초)
  App (콜드 스타트):     AX API 폴링 100ms 간격 (최대 5초)
  Finder:               딜레이 없음 (열기+리사이즈 단일 스크립트)


## 사용 API / 프레임워크

### ApplicationServices (Accessibility API)
  문서: https://developer.apple.com/documentation/accessibility/accessibility-api
  용도: 앱 윈도우 리사이즈 (AppLauncher), 글로벌 핫키 권한 확인
  - AXUIElementCreateApplication(pid)         앱의 AX 요소 참조 생성
  - AXUIElementCopyAttributeValue             포커스 윈도우(kAXFocusedWindowAttribute) 획득
  - AXUIElementSetAttributeValue              윈도우 position/size 설정
  - AXValueCreate(.cgPoint / .cgSize)         좌표/크기 값 생성
  - AXIsProcessTrusted()                      접근성 권한 확인
  - AXIsProcessTrustedWithOptions()           접근성 권한 요청 다이얼로그

### AppKit (NSWorkspace)
  용도: 앱 실행, 실행 중 앱 감지
  - NSWorkspace.shared.openApplication(at:configuration:)   앱 실행 (pid 콜백)
  - NSWorkspace.shared.runningApplications                  실행 중 앱 목록 조회

### CoreGraphics (CGEvent)
  용도: 글로벌 키보드 단축키
  - CGEvent.tapCreate()               이벤트 탭 생성 (키 입력 감시)
  - CFMachPortCreateRunLoopSource()   런루프에 이벤트 소스 등록

### AppleScript (osascript via Process)
  용도: Finder 윈도우 제어
  - Finder: "tell application Finder" → open + bounds 설정
  ※ AppLauncher, ChromeLauncher는 AX API 직접 사용 (성능 최적화)

### NSScreen
  용도: 멀티 모니터 좌표 계산, 대상 화면 결정
  - NSScreen.screens                  전체 디스플레이 목록
  - NSScreen.localizedName            디스플레이 이름 매칭
  - visibleFrame / frame              좌표 변환 (NSScreen ↔ AppleScript 좌표계)


## 리사이즈 성능 개선 이력 (2026-06-22)

### 변경 전 (AppleScript 방식)
  구조: 앱 실행 → 고정 딜레이 → Process fork(/usr/bin/osascript) → AppleScript → System Events/Chrome
  
  | 타입 | 상태 | 소요 시간 |
  |------|------|-----------|
  | App (KakaoTalk) | running | 0.71~0.75s |
  | App (PowerPoint) | cold | 4.2s |
  | URL (Translate) | running | 0.75~0.89s |
  | URL (Case) | running | 0.74~0.87s |

### 변경 후 (AX API 방식)
  구조: 앱 실행 → AX API 폴링(50ms) → 윈도우 감지 즉시 리사이즈

  | 타입 | 상태 | 소요 시간 | 개선폭 |
  |------|------|-----------|--------|
  | App (KakaoTalk) | running | 0.03~0.28s | **3~10배** |
  | App (PowerPoint) | running | 0.03~0.07s | **10배+** |
  | URL (Translate) | running | 0.34~0.44s | **약 2배** |
  | URL (Case) | running | 0.34~0.52s | **약 40%** |

### 수정한 것
  1. AppLauncher: Process+osascript+SystemEvents → AXUIElementSetAttributeValue 직접 호출
  2. ChromeLauncher: Process+osascript+Chrome AppleScript → AX API 윈도우 수 비교로 새 윈도우 감지
  3. LauncherUtils: setPosition/setSize/axApplyBounds 공용 함수로 통합
  4. 폴링 간격: 고정 딜레이 배열(0.5~2.0초) → 50ms 간격 폴링
  5. Accessibility 체크: ChromeLauncher에도 추가
  6. pid 캐싱: Chrome running 시 매 폴링마다 runningApplications 조회 제거

### 남은 병목 (제어 불가)
  - App running: 150ms activate 대기 (macOS 포커스 전환 시간)
  - URL: Chrome 윈도우 생성 시간 (0.3~0.5s, Chrome 내부 처리)


## 권한 요구사항

1. 접근성 (Accessibility): 글로벌 단축키 (CGEvent tap)
   - 앱 시작 시 2초 간격 재시도, 30초 후 alert
   - 허용 후 앱 재시작 필요 → Restart 메뉴 제공

2. 자동화 (Automation): Chrome/Finder AppleScript 제어
   - osascript 실행 시 시스템이 자동 다이얼로그 표시 (서명된 앱)
   - 서명 없는 앱: xattr -cr 후 수동 설정 필요


## 배포 현황

- 코드 서명: 미등록 (Apple Developer $99 필요)
- 현재 배포 방식: 소스 clone → Xcode 빌드 (권한 정상 동작)
- zip/dmg 배포: xattr 필요 + 수동 권한 설정 (서명 없는 앱 한계)
