Chap Project Structure
======================

## 앱 소개

Chap — 키보드 단축키 한 번으로 웹사이트·앱·폴더를 원하는 화면, 원하는 크기로 즉시 여는 macOS 메뉴바 런처.


## 프로젝트 구조

Sources/
├── Chap/                          ← 앱 타겟
│   ├── AppDelegate.swift          ← 라이프사이클, 메뉴, 핫키 (CGEvent tap)
│   ├── main.swift                 ← 엔트리포인트
│   ├── Launchers/                 ← 런처별 실행 로직
│   │   ├── ChromeLauncher.swift   ← URL (Chrome --app 모드)
│   │   ├── AppLauncher.swift      ← App (System Events 리사이즈)
│   │   ├── FinderLauncher.swift   ← Finder (단일 AppleScript)
│   │   ├── ShellLauncher.swift    ← Shell (스크립트 실행)
│   │   ├── LauncherUtils.swift    ← 공용 retryResize, showAlert
│   │   └── ResizeLogger.swift     ← CSV 성능 로그
│   └── Views/                     ← UI
│       ├── DesignSystem.swift     ← 색상, 폰트, 컴포넌트 스타일
│       ├── Components.swift       ← 공용 뷰 (CardSection, InputField 등)
│       ├── SettingsView.swift     ← 설정 메인 뷰
│       ├── SiteConfigView.swift   ← 사이트별 설정
│       └── WelcomeView.swift      ← 첫 실행 가이드
├── ChapCore/                      ← 코어 로직 (테스트 대상)
│   ├── Models.swift               ← Site, Config, LaunchType, Defaults
│   ├── SettingsViewModel.swift    ← 설정 상태 관리
│   └── Validation.swift           ← 도메인 검증, 좌표 계산
Tests/ChapCoreTests/               ← 23개 유닛 테스트
    ├── ModelTests.swift
    ├── SettingsViewModelTests.swift
    └── ValidationTests.swift

Config Files:
├── project.yml                    ← XcodeGen 프로젝트 정의
├── Chap.xcodeproj/                ← Xcode 프로젝트
└── .harness/                      ← AI 어시스턴트 규칙 (공유 심링크)

Runtime:
├── ~/.chap.json                   ← 사용자 설정 파일
├── ~/.chap.json.bak               ← 자동 백업
└── 프로젝트폴더/logs/              ← 리사이즈 성능 로그 (CSV)


## 핵심 기능

- 글로벌 단축키 (⌥Q, ⌥1~9, ⌥,) — CGEvent tap
- URL → Chrome --app 모드 + 리사이즈
- App → 실행 + System Events 리사이즈
- Finder → 폴더 열기 + 리사이즈 (단일 AppleScript, 딜레이 없음)
- Shell → 스크립트 실행 (리사이즈 없음)
- 멀티 모니터 지원 + 미니맵
- Edit/Save 모드 (실수 방지)
- Import (파일/Paste JSON/드래그앤드랍) + Export → ~/Downloads
- Config 자동 백업 (.bak)
- 순서 변경/삭제 시 자동 저장
- Restart / Uninstall (메뉴바/설정 File 메뉴)
- 리사이즈 성능 로그 (CSV)
- 접근성 권한 자동 재시도 (30초 제한 + alert)
- User Guide (⌘?)


## 단축키

글로벌 (앱 외부, ⌥ Option):
  ⌥Q     메뉴 열기
  ⌥1~9   사이트 직접 실행
  ⌥,     설정 열기

설정 창 내 (⌘ Command):
  ⌘1~9   사이트 선택
  ⌘E     편집 모드
  ⌘S     저장
  ⌘?     가이드


## 런처 타입별 동작

  URL:     Chrome --app 모드 실행 → osascript로 도메인 매칭 → bounds 설정
  App:     NSWorkspace.openApplication → System Events로 position/size 설정
  Finder:  단일 AppleScript로 열기+리사이즈 동시 (딜레이 없음)
  Shell:   Process로 스크립트 실행 (리사이즈 없음)

리사이즈 딜레이:
  URL (Chrome 실행 중):  0.5, 0.8, 1.2, 2.0초
  URL (콜드 스타트):     1.0, 2.0, 3.5, 5.0초
  App (실행 중):         0.2, 0.5, 0.8, 1.2, 2.0초
  App (콜드 스타트):     1.0, 2.0, 3.5, 5.0초
  Finder:               딜레이 없음


## 권한 요구사항

1. 접근성 (Accessibility): 글로벌 단축키 (CGEvent tap)
   - 앱 시작 시 2초 간격 재시도, 30초 후 alert
   - 허용 후 앱 재시작 필요 → Restart 메뉴 제공

2. 자동화 (Automation): Chrome/Finder AppleScript 제어
   - osascript 실행 시 시스템이 자동 다이얼로그 표시 (서명된 앱)
   - 서명 없는 앱: xattr -cr 후 수동 설정 필요


## 배포 현황

- 코드 서명: 미등록 (Apple Developer $99 필요)
- 현재 배포 방식: 소스 clone → Xcode 빌드 (권한 정상 동작)
- zip/dmg 배포: xattr 필요 + 수동 권한 설정 (서명 없는 앱 한계)
