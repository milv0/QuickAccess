Chap Project Structure
======================

## 앱 소개

Chap — 키보드 단축키 한 번으로 웹사이트·앱·폴더를 원하는 화면, 원하는 크기로 즉시 여는 macOS 메뉴바 런처.


## 프로젝트 구조

## 핵심 기능

- 글로벌 단축키 (⌥Q, ⌥1~9, ⌥,) — CGEvent tap
- URL → Chrome --app 모드 + 리사이즈
- App → 실행 + System Events 리사이즈
- Finder → 폴더 열기 + 리사이즈 (단일 AppleScript, 딜레이 없음)
- Shell → 스크립트 실행 (리사이즈 없음)
- 멀티 모니터 지원 + 미니맵
- Edit/Save 모드 (실수 방지)
- Import (파일/Paste JSON/드래그앤드랍) + Export → ~/Downloads
- Config 자동 백업 (.bak)
- 순서 변경/삭제 시 자동 저장
- Restart / Uninstall (메뉴바/설정 File 메뉴)
- 리사이즈 성능 로그 (CSV)
- 접근성 권한 자동 재시도 (30초 제한 + alert)
- User Guide (⌘?)


## 단축키

글로벌 (앱 외부, ⌥ Option):
  ⌥Q     메뉴 열기
  ⌥1~9   사이트 직접 실행
  ⌥,     설정 열기

설정 창 내 (⌘ Command):
  ⌘1~9   사이트 선택
  ⌘E     편집 모드
  ⌘S     저장
  ⌘?     가이드


## 런처 타입별 동작

  URL:     Chrome --app 모드 실행 → osascript로 도메인 매칭 → bounds 설정
  App:     NSWorkspace.openApplication → AX API로 position/size 직접 설정
  Finder:  단일 AppleScript로 열기+리사이즈 동시 (딜레이 없음)
  Shell:   Process로 스크립트 실행 (리사이즈 없음)

리사이즈 전략:
  URL (Chrome 실행 중):  osascript 점진적 재시도 (0.5, 0.8, 1.2, 2.0초)
  URL (콜드 스타트):     osascript 점진적 재시도 (1.0, 2.0, 3.5, 5.0초)
  App (실행 중):         AX API 폴링 50ms 간격 (최대 1초)
  App (콜드 스타트):     AX API 폴링 100ms 간격 (최대 5초)
  Finder:               딜레이 없음 (열기+리사이즈 단일 스크립트)


## 사용 API / 프레임워크

### ApplicationServices (Accessibility API)
  용도: 앱 윈도우 리사이즈 (AppLauncher), 글로벌 핫키 권한 확인
  - AXUIElementCreateApplication(pid)         앱의 AX 요소 참조 생성
  - AXUIElementCopyAttributeValue             포커스 윈도우(kAXFocusedWindowAttribute) 획득
  - AXUIElementSetAttributeValue              윈도우 position/size 설정
  - AXValueCreate(.cgPoint / .cgSize)         좌표/크기 값 생성
  - AXIsProcessTrusted()                      접근성 권한 확인
  - AXIsProcessTrustedWithOptions()           접근성 권한 요청 다이얼로그

### AppKit (NSWorkspace)
  용도: 앱 실행, 실행 중 앱 감지
  - NSWorkspace.shared.openApplication(at:configuration:)   앱 실행 (pid 콜백)
  - NSWorkspace.shared.runningApplications                  실행 중 앱 목록 조회

### CoreGraphics (CGEvent)
  용도: 글로벌 키보드 단축키
  - CGEvent.tapCreate()               이벤트 탭 생성 (키 입력 감시)
  - CFMachPortCreateRunLoopSource()   런루프에 이벤트 소스 등록

### AppleScript (osascript via Process)
  용도: Chrome 윈도우 제어, Finder 윈도우 제어
  - Chrome: "tell application Google Chrome" → bounds 설정
  - Finder: "tell application Finder" → open + bounds 설정
  ※ AppLauncher는 AppleScript 대신 AX API 직접 사용 (성능 최적화)

### NSScreen
  용도: 멀티 모니터 좌표 계산, 대상 화면 결정
  - NSScreen.screens                  전체 디스플레이 목록
  - NSScreen.localizedName            디스플레이 이름 매칭
  - visibleFrame / frame              좌표 변환 (NSScreen ↔ AppleScript 좌표계)


## 권한 요구사항

1. 접근성 (Accessibility): 글로벌 단축키 (CGEvent tap)
   - 앱 시작 시 2초 간격 재시도, 30초 후 alert
   - 허용 후 앱 재시작 필요 → Restart 메뉴 제공

2. 자동화 (Automation): Chrome/Finder AppleScript 제어
   - osascript 실행 시 시스템이 자동 다이얼로그 표시 (서명된 앱)
   - 서명 없는 앱: xattr -cr 후 수동 설정 필요


## 배포 현황

- 코드 서명: 미등록 (Apple Developer $99 필요)
- 현재 배포 방식: 소스 clone → Xcode 빌드 (권한 정상 동작)
- zip/dmg 배포: xattr 필요 + 수동 권한 설정 (서명 없는 앱 한계)
